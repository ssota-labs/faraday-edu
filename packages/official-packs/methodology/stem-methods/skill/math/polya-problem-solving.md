# Polya — four-phase problem solving

George Polya's heuristic: **Understand the problem → Devise a plan → Carry out the
plan → Look back**. It structures procedure-heavy math and proof planning without
replacing domain knowledge (Polya 1945; still standard in problem-solving courses).

## When to use

- Word problems, multi-step proofs, olympiad-style tasks
- When the learning goal is *strategy*, not a single fact
- Undergraduate math where Peer Instruction handles concepts but problems need
  a separate arc

## Unit arc

| Phase | Learner does | Block families |
|---|---|---|
| **Understand** | Restate goal, identify givens/constraints, draw if needed | **Narrative** + **spatial** (diagram/sketch family) |
| **Plan** | Choose strategy (work backward, auxiliary line, induction…) | **Check** (strategy MCQ) or **narrative** prompt; optional **spatial** sketch |
| **Carry out** | Execute the plan with feedback | **Executable** or **formalism** + **check** (numeric/symbolic) |
| **Look back** | Verify reasonableness, generalize, alternate method | **Formalism** (`Reveal`), **data** comparison, **check** (reflection MCQ) |

## Rules

- Understanding phase must produce a *learner artifact* (sketch, restatement),
  not passive reading.
- Plan phase: wrong strategies as distractors should be **common real choices**,
  not nonsense.
- Look back is not optional garnish — at least one "what if we changed…?" moment
  (**data** or **manipulative** variation).

## Anti-patterns

- Skipping Understand/Plan and jumping to template drills.
- Look back = "the answer was 42" with no structural reflection.

# CRA / CPA — Concrete → Representational → Abstract

**Concrete–Representational–Abstract** (CRA; Singapore **CPA** — Concrete–Pictorial–
Abstract) meets each concept three times in order: manipulate a thing, see it as a
diagram, then meet the symbol. The symbol is a *compression* of experience, not
the introduction (Bruner 1966; meta-analyses in math intervention, e.g. Bouck et
al. 2018).

## When to use

- Arithmetic, fractions, early algebra, geometry intuition
- Any learner new to a symbol system for a concept they haven't physically met
- Pairs naturally with `audience` children row; this file adds **variation** across
  representations

## Unit arc

| Phase | Learner does | Block families |
|---|---|---|
| **Concrete** | Manipulate, count, compare, transform | **Manipulative** (primary), narrative hook |
| **Representational** | See the same idea as diagram, area model, graph shape | **Spatial** + **data** (light), short narrative |
| **Abstract** | Use notation with the picture still visible | **Formalism** (thin), **check** (do, not define) |

## Rules

- Never open on bare symbols; if `<TeX>` appears, it's small and adjacent to a
  spatial or manipulative anchor.
- One invariant idea per screen in concrete phase (pairs with paged layout for
  young audiences).
- Checks ask *do* ("make the areas equal"), not *recite* ("state the definition").
- Fade concrete faster for older novices within secondary — still keep a
  representational bridge.

## Anti-patterns

- Decorative manipulative that doesn't encode the math structure.
- Jumping to abstract because "they're in high school" when they're novices *to
  this topic* (expertise-reversal cuts the other way too).

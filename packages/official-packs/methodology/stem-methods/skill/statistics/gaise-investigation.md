# GAISE investigation cycle

The **GAISE** framework (ASA; revised 2016) defines statistical thinking as an
investigation: **ask questions → collect/consider data → analyze → interpret →
communicate**. It privileges context, variability, and reasoning over recipe
formulas.

## When to use

- AP Statistics, intro data science, survey courses, civic/stat literacy
- Any unit where the **question** precedes the tool choice
- General public audience + data-heavy narratives

## Unit arc

| Phase | Learner does | Block families |
|---|---|---|
| **Question** | Formulate statistical question; variables | **Narrative** context + **check** (question type) |
| **Data** | Collect, import, or examine dataset | **Data** (charts, tables), **executable** (load/filter) |
| **Analysis** | Choose and apply appropriate methods | **Data** + **executable**; **formalism** after intuition |
| **Interpret** | Answer in context; acknowledge uncertainty | **Narrative** + **check** (interpretation MCQ) |
| **Communicate** | Visual + verbal conclusion for audience | **Data** + **presentation** or **narrative** |

## Rules

- Every analysis phase ties back to the **original question** — resist tool-first
  teaching ("today we learn t-tests").
- Variability and scope of inference are explicit in Interpret.
- Communicate uses the **data** family for honest graphics (no chartjunk).

## Anti-patterns

- Recipe stats: formula → plug numbers with no question.
- Interpretation disconnected from units and context.

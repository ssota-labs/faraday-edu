# STEM methods — overview

The **creator's own methodology always wins.** This pack is the **discipline
layer** for creators who don't bring one: fifteen named methods across math,
science, engineering, computing, and statistics, mapped to Faraday **block
families** (not single blocks). Don't recite it to the creator; apply it, cite
only if asked.

## Three layers (load the one you need)

| Layer | Pack | Question |
|---|---|---|
| Universal moves | `lecture-design` | *How* should any lesson teach? (backward design, mastery gates, generative interaction, retrieval, feed-forward feedback) |
| Learner population | `audience` | *Who* is learning? (CRA, 5E, Peer Instruction, Mayer, Merrill + layout defaults) |
| **Discipline & content** | **`stem-methods`** (this pack) | *What kind of knowledge?* (problem-solving, modeling, design, code, data…) |

**Audience** picks opening register and symbol density. **Stem-methods** picks the
**unit arc** (the shape of a concept node or multi-node unit). They compose —
they don't compete. Example: undergrad physics → Peer Instruction opens the
node (`audience`); the unit still follows the modeling cycle (`stem-methods`).

## Block families (not 1:1 mapping)

Methods point at *families* of Faraday primitives — manipulative, formalism,
check, data, executable, narrative, structure. See [block-families.md](block-families.md).

## Pick a method

Use [selection.md](selection.md) — discipline × content type × goal. **One stem
method per unit**; swap only when the content type changes (e.g. a design capstone
after a modeling unit).

# Modeling Instruction — build, test, deploy, revise models

**Modeling Instruction** (Hestenes, Wells, AMTA) organizes science around
**conceptual models**: learners observe phenomena, propose relationships, test
them against data, deploy models to new cases, and revise. Normalized FCI gains
typically far exceed lecture-only instruction (Hake 1998; PhysPort summary).

## When to use

- Mechanics, fields, chemistry relationships, population dynamics — anywhere
  "law" means a model with scope limits
- Undergraduate STEM where `audience` Peer Instruction fits but units need a
  coherent modeling spine

## Unit arc

| Phase | Learner does | Block families |
|---|---|---|
| **Phenomenon** | Explore a simple system; notice patterns | **Manipulative** (primary), **data** (collect/plot) |
| **Model construction** | Propose relationship; fit/compare representations | **Data** + **formalism**; **spatial** if geometry matters |
| **Deployment** | Apply model to a *new* context | **Manipulative** or **check** (prediction before run) |
| **Revision** | Confront misfit; refine or limit the model | **Check** (ConcepTest-style), **formalism**, second **data** pass |

## Rules

- Multiple representations (graph, diagram, equation, verbal) are part of one
  model — use **data**, **spatial**, and **formalism** families together, not
  as sequential unrelated blocks.
- Deployment must be a genuinely new surface — not the same slider with different
  numbers only.
- Documented **misconceptions** drive revision-phase checks.

## Anti-patterns

- Equation-first "modeling" with no phenomenon phase.
- Models that never fail — learners must see scope limits.

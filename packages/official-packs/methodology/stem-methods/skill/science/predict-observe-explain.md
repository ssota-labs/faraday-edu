# Predict–Observe–Explain (POE)

**Predict–Observe–Explain** is a tight inquiry loop on a single phenomenon:
commit to what will happen, observe the outcome (live, sim, or demo), then
explain the discrepancy. Classic in science education (White & Gunstone 1992);
fits inside 5E **Explore** or as a standalone micro-unit.

## When to use

- One demo, one simulation, one surprising outcome
- Secondary science, museum/kiosk interactives, undergrad concept hooks
- Any unit where **commitment before evidence** matters

## Unit arc

| Phase | Learner does | Block families |
|---|---|---|
| **Predict** | Commit to outcome (no peeking) | **Check** (prediction MCQ or numeric band), **narrative** setup |
| **Observe** | Run phenomenon; record what happened | **Manipulative** or **spatial**; **data** if measuring |
| **Explain** | Reconcile prediction and observation | **Formalism** + **narrative**; optional tutor Socratic follow-up |
| **Extend** (optional) | Change one variable; repeat POE | **Manipulative** + **check** |

## Rules

- Prediction must be **locked** before observe (disable scrub-after-reveal pattern
  until commit).
- Explain phase names the *gap*, not only the correct law.
- One POE cycle = one focal variable; don't bundle whole units.

## Anti-patterns

- Observe without prediction (passive demo).
- Explain = reading the answer with no reference to what they predicted.

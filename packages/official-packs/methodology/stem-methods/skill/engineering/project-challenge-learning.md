# Project / challenge-based learning (PjBL / CBL)

**Project-based** and **challenge-based** learning center a **deliverable** or
**challenge brief** (product, report, prototype, policy) with milestones, criteria,
and often team roles. Widespread in engineering capstones and SDG-style courses
(MDPI systematic reviews on PBL/PjBL/CBL instructional design).

## When to use

- Capstone, maker courses, hackathon-shaped units, design studios
- When success = artifact + justification, not a single quiz score
- Challenge-based variants add stakeholder / SDG framing

## Unit arc

| Phase | Learner does | Block families |
|---|---|---|
| **Brief** | Parse requirements, constraints, rubric | **Narrative** + **check** (requirements MCQ) |
| **Plan** | Milestones, division of labor, risk | **Narrative** template; optional **spatial** sketch |
| **Build** | Iterative construction | **Manipulative**, **executable**, **spatial** (3D if physical) |
| **Verify** | Test against rubric / acceptance tests | **Check** (mission), **data** (metrics) |
| **Present** | Communicate trade-offs and evidence | **Narrative** + **data**/**spatial** evidence |

## Rules

- Rubric criteria visible before build — backward design alignment.
- Milestones gate progression (`lecture-design` mastery gates on *demonstrated*
  capability, not time).
- Realistic **data** from tests beats self-report alone.

## Anti-patterns

- Project = decorate a poster about a topic with no engineering judgment.
- Challenge with no verifiable acceptance criteria.

## PBL vs PjBL vs CBL (quick)

| | Driver | End state |
|---|---|---|
| **PBL** | Ill-structured problem | Understanding + solution path |
| **PjBL** | Extended project brief | Deliverable over weeks |
| **CBL** | Real-world challenge / stakeholder | Actionable outcome + reflection |

Pick one spine per unit; don't stack all three labels on the same node.

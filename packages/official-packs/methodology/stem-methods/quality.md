# Pack `stem-methods` — quality bar

Design-quality rules for STEM lessons built with this pack. These grade the
*discipline fit* of the teaching, not the code.

- **Discipline-fit.** The chosen method matches the content type (procedural,
  conceptual, modeling, design, data, or code) per [selection.md](skill/selection.md).
- **One arc.** A single stem method leads the unit structure; audience chooses
  delivery register; lecture-design moves run underneath — no mechanical stacking
  of every method on every node.
- **Family mapping, not checkbox blocks.** Each phase uses the right *block
  family* (manipulative, formalism, check, etc.) per [block-families.md](skill/block-families.md);
  empty ornament blocks don't count.
- **Misconception-aware (science/math).** Concept checks use documented
  field misconceptions as distractors, not filler wrong answers.
- **Representation match.** Spatial content uses spatial families; processes use
  executable or manipulative families; relationships use data families — matched
  to the *content*, not to a claimed learning style.

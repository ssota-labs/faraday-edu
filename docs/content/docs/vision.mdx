## Vision

Faraday helps teachers turn existing course materials into interactive textbooks
with grounded AI tutors. Authors work with a coding agent: drop in lesson
content, and the Faraday CLI scaffolds a vinext + React lesson pinned to the
`@faraday-academy/*` runtime.

Deeper product and architecture notes live in the project-owned
[`docs/VISION.md`](../../../VISION.md) (Korean). That file is the historical
source of truth for Stage 1/2 direction; planning artifacts under this handbook
should stay aligned with it.

## Principles

1. Intent precedes implementation — PRD → story → spec/ADR → plan → code.
2. Seed + gate: maximum authoring freedom, locked runtime contracts and checks.
3. Stable IDs connect planning artifacts without coupling them to filenames.
4. Human review decides product quality; automation checks structural integrity.
5. Documentation and agent-readable Markdown are two views of the same source.

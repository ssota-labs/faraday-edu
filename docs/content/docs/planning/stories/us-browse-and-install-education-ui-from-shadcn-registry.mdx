As a lesson author or agent, I want to browse education UI components and add
only what a lesson needs so that 3D STEM lessons share consistent controls and
readouts without depending on a published npm component library.

## Context

UI distribution follows the shadcn model: source components live in the monorepo,
are documented in an `apps/` site, and are copied into consumer projects via
registry install. This replaces `@faraday-academy/ui` npm publish as the product
path.

## Acceptance criteria

- An `apps/` surface presents education components (catalog / docs style).
- Components relevant to 3D STEM lessons (controls, readouts, overlays that do
  not break fullscreen immersion) are registry-installable.
- Install path does not require `npm install @faraday-academy/ui`.
- The `3d-stem` skill documents when and how to pull registry components during
  authoring.

## Out of scope

- Building a general-purpose design system unrelated to STEM lessons
- Hosting a commercial component marketplace
- Requiring every lesson to install the full component set

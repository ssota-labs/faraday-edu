# Lecture design — overview

The **creator's own methodology always wins.** If they teach with a method (their
sequence, their assessment style, a school framework), follow it. This library is
the **default** for creators who don't bring one: a compact, evidence-based
methodology mapped to Faraday's levers. Don't recite it to the creator; apply it,
cite it only if asked.

## The default in five moves (load the one you need)

1. [backward-design.md](backward-design.md) — outcomes → evidence → activities;
   keep every lesson/check aligned to an outcome verb.
2. [prerequisite-graph.md](prerequisite-graph.md) — order by real dependencies;
   gate on demonstrated mastery, not exposure.
3. [generative-learning.md](generative-learning.md) — make the learner predict,
   construct, explain (ICAP); worked-examples-and-fade for novices.
4. [spaced-retrieval.md](spaced-retrieval.md) — quizzes are learning events;
   space, retrieve, interleave.
5. [feedback.md](feedback.md) — where-am-I / how / where-next, not just
   right/wrong.

Then: [anti-patterns.md](anti-patterns.md) (commonly believed, poorly supported)
and [methods.md](methods.md) (named methodologies per learner population).

## Audience layer

The five moves are audience-independent. *How the concept is handed to the
learner* — register, symbol density, which interaction leads — is the audience
layer: one default methodology per learner population (see [methods.md](methods.md)),
applied on top of these moves.

## Using this with a creator

- **They have a methodology** → their structure/assessment style leads; borrow
  from here only where they're silent (e.g., a sequence but no checks → add
  retrieval gates).
- **They don't** → apply the five moves and *show* the result (roadmap, a sample
  lesson with its pretest/quiz) rather than lecturing pedagogy. If they push back
  on a default ("no gates, keep it open"), their call wins — note the trade-off
  once, then follow it.

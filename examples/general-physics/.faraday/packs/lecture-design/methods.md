# Named methodologies — one per learner population

The five moves in this pack are audience-independent. *How* you hand the concept
to the learner is the audience layer: pick **one** default methodology for the
learner population and let it lead the register, symbol density, and which
interaction opens. (The base skill's `audience.md` is the source of truth for
audience gating — this file is the method summary.)

| Population | Default method | In one line |
|---|---|---|
| **Children** | **CRA** (Concrete → Representational → Abstract) | Start with a manipulable concrete model, move to a picture/diagram, then to the symbol — never open on the abstraction. |
| **Secondary** | **5E** (Engage, Explore, Explain, Elaborate, Evaluate) | Hook → let them mess with the phenomenon → name it → stretch to a new case → check. Exploration precedes the explanation. |
| **Undergraduate** | **Peer Instruction** (Mazur) | Concept question → commit an answer → reason/discuss → re-answer. Build lessons around discriminating concept checks, not derivations. |
| **General public** | **Mayer's multimedia principles** | Segmented, learner-paced, key element cued, decorative material cut. Lean and coherent over comprehensive. |
| **Professionals** | **Merrill's First Principles** | Task-centered: a real problem up front, activate prior experience, demonstrate, let them apply, then integrate into their work. |

## Rules

- **One methodology per lesson/population.** Don't blend 5E and Peer Instruction
  in the same unit — pick by who's learning.
- **Method chooses the opening interaction.** CRA opens on a concrete manipulative;
  5E opens on exploration; Peer Instruction opens on a concept question.
- **Extending this pack:** add a new methodology as its own file here (e.g.
  `methods/udl.md`) and a row above. That's the "add a teaching method" flow.
